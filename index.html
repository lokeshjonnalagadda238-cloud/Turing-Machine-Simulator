Formal Definition

A Turing Machine is a 7-tuple:

M = (Q, Σ, Γ, δ, q₀, q_acc, q_rej)

Q   = finite set of states
Σ   = input alphabet (no blank)
Γ   = tape alphabet (Σ ⊆ Γ, ⬜ ∈ Γ)
δ   = Q × Γ → Q × Γ × {L,R}
q₀ = start state
q_acc = accept state
q_rej = reject state

Church–Turing Thesis

Any effectively computable function can be computed by a Turing Machine. This defines the boundary of what algorithms can ever solve. Every modern computer is equivalent to a TM in computational power.

Configuration

A configuration is a triple (state, tape content, head position) describing the complete machine state at any moment. A computation is a sequence of configurations connected by valid transitions via δ.

Decidability

A language is decidable if a TM always halts (accept or reject). It is recognizable if a TM accepts members but may loop on non-members. The Halting Problem is undecidable — proven by diagonalization.

Instantaneous Description (ID)

An ID is written as: αqβ where α = tape left of head, q = current state, β = tape from head rightward. Each step of the TM moves from one ID to the next using δ.

Complexity Classes

Time/space on a TM defines: P (poly-time decidable), NP (poly-time verifiable), PSPACE, EXPTIME. The open question P vs NP asks if every NP problem is also in P.
